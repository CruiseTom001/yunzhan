## 高可用架构设计原则

### 可用性等级对照表

| 可用性 | 全年停机时间 |
|--------|-------------|
| 99%（两个9） | 3.65 天 |
| 99.9%（三个9） | 8.76 小时 |
| 99.99%（四个9） | 52.6 分钟 |
| 99.999%（五个9） | 5.26 分钟 |

### CAP 理论

- C（Consistency）一致性
- A（Availability）可用性
- P（Partition Tolerance）分区容忍性

### N+1 冗余

N+0 → N+1 → N+2 → 2N → 2N+1

### 故障域隔离

Region > AZ > Rack > Host > Process

### 高可用架构设计 Checklist

- 消除所有单点故障（SPOF）
- 核心服务至少 N+1 冗余
- 跨可用区部署关键组件
- 自动故障检测与切换
- 熔断器防止级联故障
