## 1.1 DevOps 概述

DevOps 是 Development（开发）和 Operations（运维）的组合。

### CALMS 模型

| 维度 | 含义 | 实践 |
|------|------|------|
| Culture（文化） | 协作与信任 | 跨职能团队、信息透明 |
| Automation（自动化） | 一切可自动化的都自动化 | CI/CD、IaC |
| Lean（精益） | 消除浪费，小批量交付 | 限制 WIP、持续改进 |
| Measurement（度量） | 数据驱动决策 | DORA 指标、SLI/SLO |
| Sharing（分享） | 知识共享与协作 | 内部开源、技术分享 |

### DevOps 的三大原则

1. 流动原则（Flow）
2. 反馈原则（Feedback）
3. 持续学习原则（Learning）
