## 2.1 什么是操作系统

操作系统（Operating System, OS）是管理计算机硬件与软件资源的系统软件，它是用户与硬件之间的桥梁。

### 操作系统的核心职责

| 职责 | 说明 |
|------|------|
| 进程管理 | 创建、调度、终止进程 |
| 内存管理 | 分配和回收内存，虚拟内存 |
| 文件系统 | 组织和管理磁盘上的数据 |
| 设备管理 | 驱动和管理硬件设备 |
| 网络管理 | 提供网络通信能力 |
| 安全管理 | 权限控制、访问控制 |

## 2.2 内核（Kernel）

内核是操作系统的核心组件，拥有对硬件的完全控制权。

### 内核的功能

\`\`\`
┌─────────────────────────────────────┐
│           用户空间 (User Space)       │
│  应用程序 → 系统调用 → 内核          │
├─────────────────────────────────────┤
│           内核空间 (Kernel Space)     │
│  进程调度 | 内存管理 | 文件系统       │
│  网络协议栈 | 设备驱动               │
├─────────────────────────────────────┤
│           硬件 (Hardware)            │
│  CPU | 内存 | 磁盘 | 网卡           │
└─────────────────────────────────────┘
\`\`\`

### 内核类型

| 类型 | 说明 | 优缺点 |
|------|------|--------|
| 宏内核 | 所有功能运行在内核态 | 性能好，但代码庞大，一个模块崩溃可能影响整个系统 |
| 微内核 | 只保留最核心功能，其他作为用户进程 | 稳定性好，但性能开销大（频繁切换） |
| 混合内核 | 宏内核 + 微内核的折中 | Linux 本质是宏内核，但支持可加载模块 |

> **Linux 是宏内核**，但通过**可加载内核模块（LKM）**机制实现了类似微内核的灵活性。

### 查看内核信息

\`\`\`bash
uname -r                       # 查看内核版本
uname -a                       # 查看完整内核信息
cat /proc/version              # 查看内核编译信息
lsmod                          # 查看已加载的内核模块
modinfo <module_name>          # 查看模块详细信息
\`\`\`

## 2.3 用户态与内核态

CPU 在不同权限级别下运行，Linux 使用 **ring 0-3** 四个级别：

\`\`\`
Ring 0 (内核态) ─── 最高权限，可执行所有指令，访问所有硬件
Ring 1          ─── 未使用
Ring 2          ─── 未使用
Ring 3 (用户态) ─── 最低权限，受限指令，不能直接访问硬件
\`\`\`

### 用户态 → 内核态的切换时机

| 场景 | 示例 |
|------|------|
| 系统调用 | 程序调用 read()、write() 等 |
| 异常 | 除零错误、缺页异常 |
| 中断 | 网卡收到数据、定时器触发 |
| 硬件中断 | 键盘输入、磁盘 I/O 完成 |

### 切换开销

用户态与内核态的切换是有开销的：
- 保存/恢复寄存器
- 切换地址空间（TLB 刷新）
- 栈切换
- 安全检查

> 这也是为什么系统调用比函数调用慢得多——一次系统调用可能需要数百到数千个 CPU 周期。

## 2.4 系统调用（System Call）

系统调用是用户程序请求内核服务的唯一合法接口。

### 常用系统调用分类

| 类别 | 系统调用 | 说明 |
|------|----------|------|
| 进程控制 | fork, exec, exit, wait | 创建/执行/退出/等待进程 |
| 文件操作 | open, read, write, close | 打开/读/写/关闭文件 |
| 设备管理 | ioctl, mmap | 设备控制/内存映射 |
| 信息维护 | getpid, alarm, sleep | 获取进程信息/定时器 |
| 通信 | pipe, shmget, mmap | 管道/共享内存/内存映射 |
| 网络 | socket, bind, listen, accept | 网络通信 |

### 使用 strace 跟踪系统调用

\`\`\`bash
# 跟踪程序的所有系统调用
strace ls

# 只跟踪文件相关系统调用
strace -e trace=file ls

# 跟踪正在运行的进程
strace -p <PID>

# 统计系统调用次数和时间
strace -c ls
\`\`\`

### 系统调用示例

\`\`\`bash
# 用 strace 观察一个简单的文件读取
strace cat /etc/hostname 2>&1 | head -20

# 输出示例：
# openat(AT_FDCWD, "/etc/hostname", O_RDONLY) = 3
# read(3, "myserver\\n", 131072) = 9
# write(1, "myserver\\n", 9) = 9
# close(3) = 0
\`\`\`

## 2.5 中断与异常

### 中断（Interrupt）

中断是硬件或软件向 CPU 发出的信号，要求 CPU 暂停当前工作来处理紧急事件。

| 类型 | 来源 | 示例 |
|------|------|------|
| 硬件中断 | 外部设备 | 网卡收到数据、键盘输入、定时器 |
| 软件中断 | 程序主动触发 | 系统调用（int 0x80 / syscall 指令） |

### 中断处理流程

\`\`\`
1. 设备发出中断信号
2. CPU 完成当前指令
3. 保存当前上下文（寄存器、程序计数器）
4. 查找中断向量表，跳转到中断处理程序
5. 执行中断处理（上半部：紧急操作）
6. 软中断/任务队列处理（下半部：可延迟操作）
7. 恢复上下文，继续执行被中断的程序
\`\`\`

### 异常（Exception）

异常是 CPU 执行指令时检测到的错误条件：

| 异常类型 | 说明 | 示例 |
|----------|------|------|
| 故障（Fault） | 可恢复的错误 | 缺页异常（Page Fault） |
| 陷阱（Trap） | 主动触发的异常 | 断点调试 |
| 终止（Abort） | 不可恢复的错误 | 硬件故障 |

> **缺页异常**是最常见的异常——当进程访问的虚拟地址不在物理内存中时触发，内核负责将所需页面从磁盘加载到内存。

## 2.6 Linux 发行版

| 发行版家族 | 代表 | 特点 |
|-----------|------|------|
| Debian 系 | Ubuntu, Debian | 包管理 apt，社区活跃，桌面友好 |
| Red Hat 系 | RHEL, CentOS, Rocky, AlmaLinux | 包管理 yum/dnf，企业级支持 |
| Arch 系 | Arch, Manjaro | 滚动更新，高度自定义 |
| SUSE 系 | openSUSE, SLES | YaST 管理工具，欧洲企业常用 |

### 运维常用发行版选择

\`\`\`
生产服务器 → Rocky Linux / AlmaLinux / Ubuntu LTS
开发环境   → Ubuntu / Fedora
容器基础   → Alpine / Debian Slim
桌面使用   → Ubuntu / Fedora
\`\`\`
