# 云栈项目开发与编码规范

本文件是本仓库的强制工程约束。所有分析、编码、测试、发布和代码审查工作开始前都必须先阅读并遵守。

## 1. 规范适用范围

- 本项目技术栈是 Vue 3、TypeScript、Pinia、Vite、Electron，不得生搬硬套仅适用于 Java 的条款。
- 采用阿里巴巴开发手册的强制原则：清晰命名、单一职责、显式边界、异常闭环、安全默认、可测试、可回滚、禁止魔法值和禁止绕过检查。
- 前端代码同时遵守项目 ESLint、TypeScript、Vue SFC 及现有架构约定；规则冲突时，以更严格且不破坏现有行为的一方为准。
- `src/`、`electron/`、`scripts/` 和构建发布配置均在本规范范围内。

## 2. 修改前要求

- 先运行 `git status -sb`，识别并保留用户已有改动，禁止擅自回滚或覆盖。
- 先阅读目标模块、关联类型、调用方、持久化结构和现有测试，不得只改报错行。
- 变更学习进度、实验判题、路由、Electron IPC 或发布配置时，必须评估旧数据兼容和桌面升级影响。
- 新增生产依赖前必须说明必要性、安全边界和维护成本；能复用现有能力时不得重复引入。

## 3. TypeScript 与 Vue 强制规则

- 禁止显式 `any`、`@ts-ignore`、`@ts-nocheck` 和无理由的类型断言；外部数据必须以 `unknown` 接收并通过类型守卫收窄。
- 变量、函数使用语义明确的 lowerCamelCase；类型、接口、Vue 组件使用 PascalCase；常量使用可读名称，跨模块常量使用 UPPER_SNAKE_CASE。
- 函数保持单一职责，优先早返回，避免超过三层嵌套；复杂分支拆成具名函数。
- 不得直接修改 props，不得在计算属性中产生副作用，不得依赖不稳定的数组下标作为业务身份。
- 共享状态必须进入 Pinia 或明确的服务模块；组件卸载时清理自行注册的监听器、定时器和外部实例。
- 异步流程必须覆盖加载、成功、空结果和失败状态；禁止静默吞掉会影响用户结果的异常。
- 用户可见文本使用简洁中文；代码、标识符和技术协议保持英文；文件使用 UTF-8。

## 4. 安全强制规则

- API Key、Token、密码和密钥不得写入源码、Git、日志、URL、`localStorage` 或普通 JSON 文件。
- 所有外部 JSON、AI 返回、导入文件和 IPC 参数都必须验证类型、大小和允许值。
- `v-html` 只能渲染经过统一 DOMPurify 清洗的内容，禁止组件自行拼接未清洗 HTML。
- Electron 必须保持 `contextIsolation: true`、`nodeIntegration: false`、`sandbox: true`；IPC 通道和命令必须使用显式白名单。
- 运行系统命令必须使用 `execFile` 和参数数组，禁止字符串拼接后交给 shell，禁止扩大破坏性命令权限。
- 删除、覆盖、导入和清空进度前必须创建可恢复备份；升级不得改变 `appId` 或默认清除 `userData`。

## 5. 课程、终端与实验规则

- 课程命令必须在训练沙箱中有可解释、与真实 Linux 语义一致的输出；不支持时明确提示“模拟”，不得伪装真实执行。
- 实验只能根据实验开始后真实执行的成功命令判定；禁止预写命令历史、模糊包含过关或用无关输出过关。
- 新增或删除课程章节时同步更新 `chapterCounts`，构建时必须通过章节数量对账。
- 题目、解释和实验步骤必须事实一致；命令、参数、输出字段和大小写需要人工复核。

## 6. UI 与可访问性规则

- 页面必须在桌面和移动视口无横向溢出、遮挡和不可达控件；浮层必须可关闭、可最小化并限制在视口内。
- 图标按钮提供可访问名称或 `title`；表单必须有明确标签、错误提示和禁用状态。
- 不添加仅作装饰的卡片、渐变球或占位功能；优先完成真实工作流。
- 颜色不能是唯一状态信号，重要状态同时使用文字或图标表达。

## 7. 强制质量门禁

每次代码修改完成后必须运行：

1. `npm run policy:check`
2. `npm run check`
3. `npm run lint`
4. `npm test`
5. `npm run build`
6. `git diff --check`

修改依赖时额外运行 `npm audit --omit=dev`。任何必需检查失败时不得宣布完成、提交、推送或生成正式安装包。

## 8. 交付与 Git 规则

- 只修改当前任务需要的文件，禁止夹带无关重构、格式化和生成物。
- 提交前检查变更清单，安装包和 `release/` 产物不进入 Git；可复现的图标与构建脚本可以提交。
- 最终说明必须包含：行为变化、验证命令、未执行检查、兼容性影响和遗留风险。
- 若必须偏离本规范，先说明原因、影响和替代保护措施，并取得用户明确同意。
