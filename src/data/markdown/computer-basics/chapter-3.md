## 4.1 内存管理概述

操作系统负责管理计算机的内存资源，为每个进程提供独立的地址空间。

### 内存管理的核心目标

| 目标 | 说明 |
|------|------|
| 隔离 | 每个进程有独立地址空间，互不干扰 |
| 抽象 | 提供虚拟内存抽象，进程无需关心物理地址 |
| 保护 | 防止进程越界访问其他进程的内存 |
| 共享 | 允许进程间安全地共享内存区域 |
| 高效 | 最大化内存利用率，减少浪费 |

## 4.2 虚拟内存

虚拟内存是操作系统提供的一种内存抽象，让每个进程以为自己拥有连续的、独立的内存空间。

### 为什么需要虚拟内存

\`\`\`
没有虚拟内存：
  进程A 直接访问物理地址 0x1000
  进程B 也直接访问物理地址 0x1000 → 冲突！

有虚拟内存：
  进程A 的虚拟地址 0x1000 → 物理地址 0x5000
  进程B 的虚拟地址 0x1000 → 物理地址 0x8000 → 不冲突！
\`\`\`

### 虚拟内存的优势

| 优势 | 说明 |
|------|------|
| 进程隔离 | 每个进程独立地址空间 |
| 简化链接 | 程序总是从相同虚拟地址开始 |
| 简化加载 | 通过页表映射实现加载 |
| 简化共享 | 不同进程的虚拟页映射到同一物理页 |
| 内存保护 | 页表项中设置权限位 |

### 虚拟地址转换

\`\`\`
CPU 发出虚拟地址
    │
    ▼
┌─────────────┐    命中
│  TLB 缓存    │──────────► 物理地址
└──────┬──────┘
       │ 未命中
       ▼
┌─────────────┐    找到映射
│  页表        │──────────► 物理地址
└──────┬──────┘
       │ 页不在内存
       ▼
┌─────────────┐
│ 缺页异常处理  │ → 从磁盘加载页面到内存
└─────────────┘
\`\`\`

## 4.3 分页机制

### 基本概念

| 概念 | 说明 |
|------|------|
| 页（Page） | 虚拟内存的固定大小块，通常 4KB |
| 页框（Frame） | 物理内存的固定大小块，与页大小相同 |
| 页表 | 存储虚拟页到物理页框映射的数据结构 |
| TLB | 页表缓存，加速地址转换 |

### 页表项（PTE）结构

\`\`\`
┌──────┬─────┬──────┬──────┬───────┬──────┐
│ 物理页号 │ 存在位 │ 读写位 │ 用户位 │ 访问位 │ 脏位  │
└──────┴─────┴──────┴──────┴───────┴──────┘
  存在位(P)：页面是否在物理内存中
  读写位(R/W)：读写权限
  用户位(U/S)：用户态/内核态可访问
  访问位(A)：页面是否被访问过（用于页面置换算法）
  脏位(D)：页面是否被修改过
\`\`\`

### 多级页表

64 位系统地址空间巨大（2^64），单级页表不现实，Linux 使用**四级页表**：

\`\`\`
虚拟地址分解：
┌────────┬────────┬────────┬────────┬──────────┐
│ PGD索引 │ PUD索引 │ PMD索引 │ PTE索引 │ 页内偏移   │
└────────┴────────┴────────┴────────┴──────────┘

查找过程：
PGD → PUD → PMD → PTE → 物理页框 + 偏移 → 物理地址
\`\`\`

### 查看内存页面信息

\`\`\`bash
# 查看页面大小
getconf PAGESIZE              # 通常 4096 (4KB)

# 查看进程的内存映射
pmap <PID>                    # 查看进程内存映射
pmap -x <PID>                 # 详细信息
cat /proc/<PID>/maps          # 查看虚拟内存区域
cat /proc/<PID>/smaps         # 更详细的内存映射信息
\`\`\`

## 4.4 Swap 交换空间

当物理内存不足时，操作系统将不活跃的内存页交换到磁盘的 Swap 空间。

### Swap 配置

\`\`\`bash
# 查看 Swap 信息
free -h
swapon --show

# 创建 Swap 文件
dd if=/dev/zero of=/swapfile bs=1G count=4   # 创建 4G 文件
chmod 600 /swapfile                           # 设置权限
mkswap /swapfile                              # 格式化为 swap
swapon /swapfile                              # 启用 swap

# 永久生效（写入 fstab）
echo '/swapfile none swap sw 0 0' >> /etc/fstab

# 关闭 swap
swapoff /swapfile
\`\`\`

### swappiness 参数

\`\`\`bash
# swappiness 控制内核使用 swap 的积极程度
# 范围 0-100，默认 60
# 值越低，越倾向于保留内存中的数据
# 值越高，越积极使用 swap

cat /proc/sys/vm/swappiness    # 查看当前值
sysctl vm.swappiness=10        # 临时修改
echo 'vm.swappiness=10' >> /etc/sysctl.conf  # 永久修改

# 推荐值：
# 桌面系统：10-30（减少卡顿）
# 服务器：10-30（数据库等对延迟敏感的应用）
# 不推荐设为 0（内核仍可能在极端情况下使用 swap）
\`\`\`

### Swap 使用场景

| 场景 | 建议 |
|------|------|
| 物理内存充足 | Swap 可小一些（等于或小于内存） |
| 物理内存不足 | Swap 应大于内存（如内存的 1-2 倍） |
| 数据库服务器 | Swap 小一些，swappiness 设低 |
| 桌面系统 | 保留一定 Swap，防止 OOM |

## 4.5 内存映射（mmap）

mmap 将文件或设备映射到进程的虚拟地址空间，实现文件 I/O 的零拷贝访问。

### mmap 的工作原理

\`\`\`
传统文件读取：
  磁盘 → 内核缓冲区 → 用户缓冲区 → 应用程序（2次拷贝）

mmap 方式：
  磁盘 → 内核缓冲区 → 映射到用户地址空间 → 应用程序（0次额外拷贝）
\`\`\`

### mmap 的应用场景

| 场景 | 说明 |
|------|------|
| 大文件处理 | 避免一次性将整个文件读入内存 |
| 共享内存 | 多个进程映射同一文件实现共享 |
| 动态链接库 | .so 文件通过 mmap 加载 |
| 数据库 | 数据库文件的高效访问 |

\`\`\`bash
# 查看进程的内存映射
cat /proc/<PID>/maps

# 输出示例：
# 地址范围              权限  偏移     设备   inode  路径
# 00400000-0040d000     r-xp  00000000 fd:00 12345  /usr/bin/cat
# 7f8a00000000-7f8a00200000 r-xp 00000000 fd:00 67890 /lib/x86_64-linux-gnu/libc.so
\`\`\`

## 4.6 OOM Killer

当系统内存耗尽时，Linux 内核的 OOM Killer 会选择一个进程终止以释放内存。

### OOM Score 计算

\`\`\`bash
# 查看进程的 OOM Score
cat /proc/<PID>/oom_score

# 调整 OOM Score（-1000 到 1000，越高越容易被杀）
echo -500 > /proc/<PID>/oom_score_adj   # 降低被杀概率
echo 500 > /proc/<PID>/oom_score_adj    # 提高被杀概率

# 完全禁止 OOM Killer 杀死某进程
echo -1000 > /proc/<PID>/oom_score_adj

# 查看最近的 OOM 事件
dmesg | grep -i "oom"
journalctl -k | grep -i "oom"
\`\`\`

### 防止 OOM 的策略

\`\`\`bash
# 1. 合理配置 Swap
# 2. 设置内存限制（cgroup）
# 3. 监控内存使用
# 4. 调整关键进程的 oom_score_adj
# 5. 完全禁用 OOM Killer（不推荐）
echo 0 > /proc/sys/vm/oom_kill_allocating_task
\`\`\`
