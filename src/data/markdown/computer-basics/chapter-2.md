## 3.1 进程基本概念

**进程（Process）**是程序在操作系统中的执行实例，是资源分配的基本单位。

### 进程的组成

\`\`\`
进程 = 程序代码 + 数据 + PCB（进程控制块）
\`\`\`

### PCB（进程控制块）包含的信息

| 信息 | 说明 |
|------|------|
| PID | 进程标识符，唯一标识一个进程 |
| PPID | 父进程 ID |
| 状态 | 运行/就绪/阻塞等 |
| 优先级 | 调度优先级 |
| 程序计数器 | 下一条指令地址 |
| 内存指针 | 代码段、数据段、栈的地址 |
| 上下文 | 寄存器值 |
| I/O 状态 | 打开的文件列表 |
| 记账信息 | CPU 时间、内存使用等 |

### 查看进程信息

\`\`\`bash
ps aux                        # 查看所有进程
ps -ef                        # 另一种格式
cat /proc/<PID>/status        # 查看进程详细信息
ls -la /proc/<PID>/fd         # 查看进程打开的文件描述符
\`\`\`

## 3.2 进程状态

Linux 进程有以下主要状态：

\`\`\`
                    fork()
                      │
                      ▼
              ┌──────────────┐
              │  就绪 (Ready) │ ◄──── 被调度器选中
              └──────┬───────┘
                     │ 获得CPU
                     ▼
              ┌──────────────┐
     ┌───────│  运行 (Running)│───────┐
     │       └──────────────┘       │
     │ 时间片用完                    │ 等待事件
     │                              │ (I/O/信号/锁)
     ▼                              ▼
  就绪队列                  ┌──────────────┐
                           │  阻塞 (Blocked)│
                           └──────┬───────┘
                                  │ 事件完成
                                  ▼
                               就绪队列
\`\`\`

### 内核视角的进程状态

| 状态 | 缩写 | 说明 |
|------|------|------|
| RUNNING | R | 正在运行或就绪等待调度 |
| INTERRUPTIBLE | S | 可中断睡眠（等待事件，可被信号唤醒） |
| UNINTERRUPTIBLE | D | 不可中断睡眠（等待 I/O，不可被信号唤醒） |
| STOPPED | T | 被暂停（SIGSTOP/SIGTSTP） |
| ZOMBIE | Z | 僵尸状态（已终止，等待父进程回收） |

\`\`\`bash
# 查看进程状态
ps -eo pid,ppid,stat,cmd | head

# STAT 列解读：
# S  - 可中断睡眠（大多数守护进程）
# R  - 正在运行
# D  - 不可中断睡眠（通常是 I/O 等待）
# Z  - 僵尸进程
# T  - 被停止
# <  - 高优先级
# N  - 低优先级
# +  - 前台进程组
# s  - 会话领导者
\`\`\`

## 3.3 进程创建

### fork() —— 创建子进程

\`\`\`bash
# fork 创建的子进程是父进程的副本
# 子进程获得父进程的：
# - 代码段（共享，只读）
# - 数据段、堆、栈（写时复制 Copy-on-Write）
# - 文件描述符表（共享偏移量）
# - 信号处理方式
\`\`\`

### exec() —— 执行新程序

\`\`\`bash
# exec 系列函数用新程序替换当前进程的地址空间
# 成功调用后不返回（因为进程已被替换）
# 常见组合：fork() + exec() = 创建新进程执行新程序
\`\`\`

### 进程创建流程

\`\`\`
父进程                    子进程
  │                         │
  ├─ fork() ──────────────► │ (子进程开始执行)
  │                         │
  │  (继续执行)              ├─ exec() (可选，加载新程序)
  │                         │
  ├─ wait() ◄──────────────┤ (子进程结束)
  │                         │
  │  (回收子进程状态)         └─ 僵尸 → 消失
\`\`\`

### 实际观察

\`\`\`bash
# 观察进程树
pstree -p                     # 查看进程树关系
ps -ef --forest               # 树形显示

# systemd 是现代 Linux 的 1 号进程
ps -p 1 -o pid,comm           # 查看 PID 1 的进程名
\`\`\`

## 3.4 进程调度

### 调度策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| SCHED_OTHER | 默认的 CFS 调度 | 普通进程 |
| SCHED_FIFO | 实时先进先出 | 实时任务，不主动让出则一直运行 |
| SCHED_RR | 实时轮转 | 实时任务，按时间片轮转 |

### CFS（完全公平调度器）

Linux 默认使用 CFS 调度普通进程：

- 每个进程维护一个**虚拟运行时间（vruntime）**
- 调度器总是选择 vruntime 最小的进程运行
- 优先级高的进程 vruntime 增长更慢，获得更多 CPU 时间

### 进程优先级

\`\`\`bash
# nice 值范围：-20（最高优先级）到 19（最低优先级）
# 默认 nice 值为 0

nice -n 10 ./my_script.sh     # 以低优先级运行
renice -n -5 -p 1234          # 提高运行中进程的优先级

# 实时优先级：1-99，数值越大优先级越高
chrt -f 50 ./realtime_app     # 以 SCHED_FIFO 优先级 50 运行
\`\`\`

## 3.5 信号（Signal）

信号是操作系统向进程发送的异步通知机制。

### 常用信号

| 信号 | 编号 | 默认行为 | 说明 |
|------|------|----------|------|
| SIGHUP | 1 | 终止 | 终端关闭时发送，常用于重载配置 |
| SIGINT | 2 | 终止 | Ctrl+C 产生 |
| SIGQUIT | 3 | 终止+Core | Ctrl+\\ 产生 |
| SIGKILL | 9 | 终止 | 强制终止，不可捕获、不可忽略 |
| SIGTERM | 15 | 终止 | 优雅终止，可捕获（默认 kill 信号） |
| SIGSTOP | 19 | 停止 | 暂停进程，不可捕获 |
| SIGCONT | 18 | 继续 | 恢复暂停的进程 |
| SIGUSR1 | 10 | 终止 | 用户自定义信号1 |
| SIGUSR2 | 12 | 终止 | 用户自定义信号2 |
| SIGCHLD | 17 | 忽略 | 子进程状态变化时通知父进程 |

### 信号处理方式

\`\`\`bash
# 1. 默认处理（执行信号的默认行为）
# 2. 忽略信号（SIGKILL 和 SIGSTOP 不可忽略）
# 3. 自定义处理函数（捕获信号）

# 发送信号
kill -TERM 1234               # 发送 SIGTERM
kill -HUP $(cat /var/run/nginx.pid)  # 重载 Nginx 配置
kill -9 1234                  # 强制终止（最后手段）

# 在 Shell 脚本中捕获信号
trap 'echo "Caught SIGINT"; exit 1' INT
trap 'reload_config' HUP
trap 'cleanup; exit 0' TERM INT
\`\`\`

## 3.6 僵尸进程与孤儿进程

### 僵尸进程（Zombie）

子进程已终止，但父进程尚未调用 wait() 回收其退出状态。

\`\`\`bash
# 查找僵尸进程
ps aux | grep 'Z'
ps -eo pid,ppid,stat,cmd | grep 'Z'

# 清理僵尸进程的方法：
# 1. 让父进程调用 wait()/waitpid()
# 2. 杀死父进程（僵尸会被 init 进程接管并回收）
kill -TERM <PPID>
\`\`\`

### 孤儿进程（Orphan）

父进程先于子进程终止，子进程被 init（PID 1）或 systemd 接管。

\`\`\`bash
# 孤儿进程不是问题，init 会负责回收
# 查看子进程的 PPID 是否变为 1
ps -eo pid,ppid,cmd
\`\`\`

### 预防僵尸进程

\`\`\`bash
# 方法1：父进程正确处理 SIGCHLD
# 在父进程中注册信号处理函数
trap 'wait' CHLD

# 方法2：双 fork 技巧
# 父进程 fork 子进程，子进程再 fork 孙子进程后立即退出
# 孙子进程成为孤儿，被 init 接管

# 方法3：显式忽略 SIGCHLD
# signal(SIGCHLD, SIG_IGN)
\`\`\`

## 3.7 线程

### 进程 vs 线程

| 特性 | 进程 | 线程 |
|------|------|------|
| 地址空间 | 独立 | 共享（代码段、数据段、堆） |
| 资源 | 独立（文件描述符等） | 共享 |
| 栈 | 独立栈 | 各线程有独立栈 |
| 创建开销 | 大（需要复制地址空间） | 小（共享地址空间） |
| 切换开销 | 大 | 小 |
| 通信 | 需要 IPC 机制 | 直接读写共享变量 |
| 安全性 | 一个进程崩溃不影响其他 | 一个线程崩溃可能导致整个进程崩溃 |

### Linux 线程实现

Linux 中的线程是通过**轻量级进程（LWP）**实现的，每个线程在内核中都有对应的 task_struct。

\`\`\`bash
# 查看进程的线程
ps -T -p <PID>                # 查看指定进程的线程
top -H -p <PID>               # 按线程查看
ls /proc/<PID>/task/          # 查看线程目录

# 查看线程数限制
cat /proc/<PID>/status | grep Threads
\`\`\`

### 多线程的常见问题

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| 竞态条件 | 多线程同时修改共享数据 | 互斥锁（Mutex） |
| 死锁 | 多个线程互相等待对方释放锁 | 锁排序、超时机制 |
| 内存可见性 | 一个线程的修改对其他线程不可见 | 内存屏障、volatile |

## 3.8 进程间通信（IPC）

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 管道（Pipe） | 半双工，父子进程间 | Shell 管道 |
| 命名管道（FIFO） | 半双工，任意进程间 | 进程间简单通信 |
| 消息队列 | 消息格式化传递 | 异步消息处理 |
| 共享内存 | 最快的 IPC 方式 | 大量数据交换 |
| 信号量 | 计数器，控制资源访问 | 同步与互斥 |
| Socket | 全双工，可跨网络 | 网络通信 |

\`\`\`bash
# 查看 IPC 资源
ipcs                          # 查看所有 IPC 资源
ipcs -m                       # 查看共享内存
ipcs -q                       # 查看消息队列
ipcs -s                       # 查看信号量
\`\`\`
