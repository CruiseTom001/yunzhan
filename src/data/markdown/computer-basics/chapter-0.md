## 1.1 计算机的基本组成

现代计算机遵循**冯·诺依曼体系结构**，由以下核心部件组成：

| 部件 | 功能 | 说明 |
|------|------|------|
| 运算器 | 执行算术和逻辑运算 | CPU 中的 ALU（算术逻辑单元） |
| 控制器 | 指挥协调各部件工作 | CPU 中的 CU（控制单元） |
| 存储器 | 存储程序和数据 | 内存（RAM）和外存（磁盘） |
| 输入设备 | 接收外部信息 | 键盘、鼠标、网卡 |
| 输出设备 | 输出处理结果 | 显示器、打印机、网卡 |

> 运算器 + 控制器 = **CPU（中央处理器）**

## 1.2 CPU —— 计算机的大脑

### CPU 核心参数

| 参数 | 说明 |
|------|------|
| 主频 | 时钟频率，如 3.4GHz，影响单核性能 |
| 核心数 | 物理核心数量，多核可并行处理任务 |
| 线程数 | 逻辑线程数（超线程技术可使 1 核模拟 2 线程） |
| 缓存 | L1（最快最小）、L2、L3（最慢最大），缓解 CPU-内存速度差 |
| 指令集 | x86_64、ARM 等，决定 CPU 能执行的指令 |

### CPU 指令执行周期

\`\`\`
取指(Fetch) → 译码(Decode) → 执行(Execute) → 写回(Write Back)
\`\`\`

### 查看 CPU 信息

\`\`\`bash
# Linux 查看 CPU 信息
lscpu                          # 查看 CPU 架构信息
cat /proc/cpuinfo              # 详细 CPU 信息
nproc                          # 查看逻辑 CPU 核心数

# 常用指标
# CPU 使用率 = 1 - (idle_time / total_time)
# Load Average = 运行队列中的平均进程数
\`\`\`

## 1.3 内存（RAM）

内存是 CPU 可以直接访问的存储空间，断电后数据丢失。

### 内存层次结构

\`\`\`
┌──────────────┐  最快、最小、最贵
│  CPU 寄存器   │  ~1ns
├──────────────┤
│  L1 Cache    │  ~1ns, 32-64KB
├──────────────┤
│  L2 Cache    │  ~4ns, 256KB-1MB
├──────────────┤
│  L3 Cache    │  ~12ns, 4-64MB
├──────────────┤
│  主存 (RAM)   │  ~100ns, 8-256GB
├──────────────┤
│  SSD 固态硬盘  │  ~0.1ms, 256GB-4TB
├──────────────┤
│  HDD 机械硬盘  │  ~10ms, 1-20TB     最慢、最大、最便宜
└──────────────┘
\`\`\`

### 内存关键概念

| 概念 | 说明 |
|------|------|
| 物理内存 | 实际安装的 RAM 容量 |
| 虚拟内存 | 操作系统提供的地址空间抽象，每个进程独享 |
| Swap | 将不活跃的内存页交换到磁盘，扩展可用内存 |
| 缓存 | 系统利用空闲内存缓存磁盘数据，加速读取 |
| 缓冲 | 写入磁盘前的临时存储区域 |

### 查看内存信息

\`\`\`bash
free -h                        # 查看内存使用（人类可读）
cat /proc/meminfo              # 详细内存信息
vmstat 1 5                     # 每秒查看内存和 CPU 状态，共 5 次
\`\`\`

### free 命令输出解读

\`\`\`
              total    used    free   shared  buff/cache  available
Mem:          7.8G    3.2G    1.1G    256M        3.5G       4.0G
Swap:         2.0G    0.5G    1.5G
\`\`\`

> **注意**：\`available\` 才是真正可用的内存，它包含了可以快速释放的 buff/cache。

## 1.4 磁盘存储

### HDD vs SSD

| 特性 | HDD（机械硬盘） | SSD（固态硬盘） |
|------|-----------------|-----------------|
| 原理 | 磁盘旋转 + 磁头读写 | 闪存芯片电子读写 |
| 随机 IOPS | ~100-200 | ~10,000-100,000+ |
| 顺序读写 | 100-200 MB/s | 500-7000 MB/s |
| 延迟 | 5-10ms | 0.01-0.1ms |
| 价格 | 低 | 较高 |
| 适用 | 冷数据存储、备份 | 热数据、数据库、系统盘 |

### 磁盘关键指标

| 指标 | 说明 |
|------|------|
| IOPS | 每秒读写次数（Input/Output Operations Per Second） |
| 吞吐量 | 每秒读写数据量（MB/s） |
| 延迟 | 单次 I/O 操作的响应时间 |
| 顺序读写 | 连续地址读写，速度快 |
| 随机读写 | 不连续地址读写，速度慢 |

### 查看磁盘信息

\`\`\`bash
lsblk                         # 查看块设备
fdisk -l                      # 查看磁盘分区
iostat -x 1                   # 实时查看 I/O 统计
iotop                         # 按 I/O 排序查看进程
\`\`\`

## 1.5 网络硬件基础

### 网卡（NIC）

网卡是计算机与网络连接的硬件接口。

\`\`\`bash
ip link show                   # 查看网卡列表
ethtool eth0                   # 查看网卡详细信息（速度、双工模式）
ethtool -S eth0                # 查看网卡统计信息（丢包等）
\`\`\`

### 网络带宽与延迟

| 概念 | 说明 |
|------|------|
| 带宽 | 网络最大传输速率（如 1Gbps） |
| 延迟 | 数据从源到目的地的往返时间（RTT） |
| 吞吐量 | 实际有效传输速率 |
| 丢包率 | 传输过程中丢失数据包的比例 |

### 网络拓扑中的常见设备

| 设备 | 功能 | 工作层级 |
|------|------|----------|
| 集线器（Hub） | 广播转发 | 物理层 |
| 交换机（Switch） | 基于 MAC 转发 | 数据链路层 |
| 路由器（Router） | 基于 IP 转发 | 网络层 |
| 防火墙（Firewall） | 过滤流量 | 网络层/传输层/应用层 |
| 负载均衡器 | 分发流量 | 传输层/应用层 |

## 1.6 计算机性能瓶颈分析

运维中常见的性能瓶颈：

\`\`\`
CPU 瓶颈 → Load Average 高，%usr 高
内存瓶颈 → Swap 使用多，OOM Kill
磁盘瓶颈 → %util 高，await 高，iowait 高
网络瓶颈 → 丢包、重传、带宽跑满
\`\`\`

### 快速排查思路

\`\`\`bash
# 1. CPU 是否瓶颈？
top -bn1 | head -5             # 看 load average 和 CPU 使用率

# 2. 内存是否瓶颈？
free -h                        # 看 available 和 Swap 使用

# 3. 磁盘是否瓶颈？
iostat -x 1 3                  # 看 %util 和 await

# 4. 网络是否瓶颈？
sar -n DEV 1 3                 # 看网络流量和丢包
\`\`\`
