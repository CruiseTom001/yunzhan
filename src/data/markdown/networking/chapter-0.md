## 1.1 为什么需要分层模型

网络通信极其复杂，分层模型将复杂问题分解为多个独立的层，每层只关注自己的职责，降低耦合度。

### 分层的优势

- **模块化**：每层独立开发和演进
- **标准化**：各层接口标准化，不同厂商设备可互操作
- **故障隔离**：问题可以定位到具体层
- **教学友好**：帮助理解数据在网络中的流转过程

## 1.2 OSI 七层模型

OSI（Open Systems Interconnection）是 ISO 制定的理论模型，共七层：

| 层号 | 名称 | 英文 | 核心功能 | 典型协议/设备 |
|------|------|------|----------|--------------|
| 7 | 应用层 | Application | 为用户提供网络服务接口 | HTTP, FTP, SMTP, DNS, SSH |
| 6 | 表示层 | Presentation | 数据格式转换、加密解密、压缩 | SSL/TLS, JPEG, ASCII |
| 5 | 会话层 | Session | 建立、管理、终止会话 | NetBIOS, RPC, SOCKS |
| 4 | 传输层 | Transport | 端到端可靠传输、流量控制、错误校验 | TCP, UDP |
| 3 | 网络层 | Network | 路由选择、逻辑寻址（IP 地址） | IP, ICMP, OSPF, 路由器 |
| 2 | 数据链路层 | Data Link | 帧封装、物理寻址（MAC 地址）、差错检测 | Ethernet, ARP, 交换机 |
| 1 | 物理层 | Physical | 比特流传输、电气特性、物理介质 | 网线、光纤、集线器 |

### 数据在各层的变化

\`\`\`
应用层：    [ 原始数据 ]
传输层：    [ TCP头 | 原始数据 ]                     → 段（Segment）
网络层：    [ IP头  | TCP头 | 原始数据 ]              → 包（Packet）
链路层：    [ 帧头  | IP头  | TCP头 | 原始数据 | 帧尾 ]  → 帧（Frame）
物理层：    010101110101010...                       → 比特流（Bits）
\`\`\`

## 1.3 TCP/IP 四层模型

TCP/IP 是实际互联网使用的协议栈，将 OSI 的七层简化为四层：

| TCP/IP 层 | 对应 OSI 层 | 核心协议 |
|-----------|------------|----------|
| 应用层 | 应用层 + 表示层 + 会话层 | HTTP, HTTPS, DNS, SSH, FTP, SMTP |
| 传输层 | 传输层 | TCP, UDP |
| 网络层 | 网络层 | IP, ICMP, ARP |
| 网络接口层 | 数据链路层 + 物理层 | Ethernet, Wi-Fi |

## 1.4 数据的实际传输过程

当你在浏览器中输入 \`https://example.com\` 时：

1. **应用层**：浏览器构造 HTTP 请求报文
2. **传输层 (TCP)**：将 HTTP 数据分段，添加 TCP 头（源端口、目标端口 443、序号等）
3. **网络层 (IP)**：添加 IP 头（源 IP、目标 IP、TTL 等）
4. **链路层 (Ethernet)**：添加以太网帧头和帧尾（源 MAC、目标 MAC）
5. **物理层**：将帧转为电信号/光信号传输

接收端反向处理，逐层解封装。

## 1.5 封装与解封装实战（使用 tcpdump 观察）

\`\`\`bash
# 抓取一次 HTTP 请求的完整数据包
sudo tcpdump -i eth0 -nn -X port 80 -c 10

# 只看 TCP 层信息
sudo tcpdump -i eth0 -nn 'tcp port 80'

# 观察三次握手过程
sudo tcpdump -i eth0 -nn 'tcp[tcpflags] & (tcp-syn|tcp-fin) != 0'
\`\`\`

## 1.6 常见面试问题

**Q：为什么 TCP/IP 是四层而不是七层？**
A：TCP/IP 将 OSI 的上三层（应用、表示、会话）合并为应用层，将下两层（链路、物理）合并为网络接口层，更加务实。OSI 的会话层和表示层在 TCP/IP 中没有独立实现。

**Q：路由器工作在哪一层？交换机呢？**
A：路由器工作在第三层（网络层），根据 IP 地址转发；传统交换机工作在第二层（链路层），根据 MAC 地址转发。三层交换机兼具路由功能。
