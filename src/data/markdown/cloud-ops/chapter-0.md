## 云原生架构（Cloud Native）

### 12-Factor App 方法论

| 因素 | 规则 |
|------|------|
| I. 代码库 | 一份代码库，多次部署 |
| II. 依赖 | 显式声明依赖 |
| III. 配置 | 配置与代码分离 |
| IV. 后端服务 | 后端服务作为附加资源 |
| V. 构建/发布/运行 | 严格分离三个阶段 |
| VI. 进程 | 无状态进程 |
| VII. 端口绑定 | 通过端口暴露服务 |
| VIII. 并发 | 通过进程模型扩展 |
| IX. 易处理 | 快速启动和优雅关闭 |
| X. 环境等价 | 开发与生产环境一致 |
| XI. 日志 | 日志作为事件流 |
| XII. 管理进程 | 管理任务一次性运行 |

### 优雅关闭（Graceful Shutdown）

SIGTERM 信号处理、Graceful Shutdown（30s 超时）。

### Service Mesh 概念

将通信逻辑从应用代码中剥离到 Sidecar 代理（如 Istio + Envoy）。
