# 变更日志

本文件遵循 [Keep a Changelog](https://keepachangelog.com/) 简化版,每条变更在括号内标注功能象限:
A = 必须同步(进度/实验/IPC/Schema) · B = 推荐同步(UI/内容) · C = 桌面专属 · D = 桌面跳过。

## [1.2.3] - 2026-07-23

### 新增
- 网页端每日学习记录支持从后端配置多个 AI 供应商，并在润色前选择使用。后端通过 `AI_PROVIDERS_JSON` 环境变量配置；新增 `GET /api/study-notes/ai/providers` 接口下发不含 API Key 的供应商列表；向前兼容旧的 `AI_API_KEY` 等单供应商环境变量。(Web 端 B 象限同步)

## [1.2.2] - 2026-07-23

### 新增
- 网页端每日学习记录 AI 润色支持流式传输（SSE），边生成边实时显示润色结果，降低 Vercel Function 超时风险。(Web 端 B 象限同步)

## [1.2.1] - 2026-07-22

### 修复
- 桌面端账号 API 保存请求默认超时从 15 秒提高到 45 秒，降低每日学习记录保存时因桌面代理链路等待过短导致的“服务器响应超时”。(桌面端 C 象限同步)
- 修复深色主题下原生模型下拉选项白底白字导致内容不可读的问题。(桌面端 B 象限同步)

## [1.2.0] - 2026-07-22

### 新增
- 桌面端每日学习记录支持保存多个本地 AI 供应商，并在每次润色前选择本次使用的供应商；旧版单供应商配置会自动迁移为供应商列表。(桌面端 C 象限同步)

## [1.1.6] - 2026-07-22

### 修复
- 桌面端账号 API 主进程代理改用 Electron `net.fetch`，修复 Node 网络栈直连云端超时导致无法登录的问题。(桌面端 C 象限同步)

## [1.1.5] - 2026-07-22

### 修复
- 桌面端账号 API 改由 Electron 主进程代理并维护内存会话 Cookie，修复登录后每日学习记录仍提示“请先登录”的问题。(桌面端 C 象限同步)

## [1.1.4] - 2026-07-21

### 修复
- 后端兼容 Electron file 来源的多种 Origin 形态，修复桌面端登录"请求来源未获授权"。(桌面端 C 象限同步)

## [1.1.3] - 2026-07-21

### 修复
- 修复桌面端账号服务请求头兜底注入，解决登录时报”请求来源未获授权”。(桌面端 C 象限同步)

## [1.1.2] - 2026-07-21

### 修复
- 桌面端安装包内置云栈账号服务地址，修复正式安装包登录时报”尚未配置云栈账号服务地址”的问题。(桌面端 C 象限同步)

## [1.1.1] - 2026-07-21

### 修复
- 桌面端 AI 测试连接现在会显示供应商返回的错误详情，并对 API Key 做脱敏处理。(桌面端 C 象限同步)
- 网页端每日学习记录的 AI 配置状态与云端代理请求超时处理同步更新。(桌面端 C 象限同步)
