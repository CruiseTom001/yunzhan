## 1. 交接摘要

云栈已经从一个离线课程网页，发展为同时支持 Web、Electron 桌面端和云端账号的运维学习平台。当前生产版已经具备课程学习、严格实验判题、模拟 Linux 终端、测验与复习、学习进度与证书、邮箱注册和找回密码、云端进度同步、超管用户管理等能力。

当前正式架构为：

```text
GitHub main
  -> Vercel（Vue 静态前端 + Express Serverless API）
  -> Neon PostgreSQL（账号、会话、学习进度、备份、审计）
  -> QQ SMTP（注册与密码重置验证码）
```

特别重要：工作区内还有一组没有完成、没有提交、没有上线的“个人账号中心 + 数据库登录限流 + 审计日志后台”改动。下一个 Agent 必须保留这些文件并继续完成，不能执行 `git reset --hard`、`git checkout -- .` 或删除未跟踪文件。

- 全部通过后显示绿色扫描线和部署成功反馈。

### 3.8 云端账号和进度同步

- 用户可通过邮箱验证码自行注册。
- 用户名显示保留原始大小写。
- 用户名登录和唯一性按大小写不敏感处理，避免 `Cruise` 与 `cruise` 成为重复账号。
- 用户可以使用用户名或邮箱登录。
- 登录会话使用随机不透明 Token。
- 浏览器只保存 `HttpOnly` Cookie，数据库只保存 Token 的 SHA-256 摘要。
- 学习进度按账号存储在 Neon PostgreSQL。
- 多设备保存使用乐观版本号。
- 发生 409 版本冲突时，客户端合并完成章节、学习日、实验、测验等进度后重试。
- 首次登录账号版时，可把旧版未分账号的本机进度迁移到第一个账号，且只迁移一次。
- 云端异常时继续使用本地缓存，并在顶部显示同步状态。
- 最近 20 个云端进度版本可作为业务级恢复历史。

### 3.9 邮箱注册和忘记密码

- 注册验证码使用 QQ 邮箱 SMTP 发送。
- 登录页已增加“忘记密码”。
- 密码重置要求验证已绑定邮箱和验证码。
- 注册与密码重置使用不同 purpose，验证码不能跨用途复用。
- 验证码有效期 10 分钟。
- 同一邮箱 60 秒冷却、每小时最多 5 次。
- 同一 IP 每小时最多 20 次验证码请求。
- 单个验证码最多尝试 5 次。
- 验证码只以 HMAC 摘要存储，不保存明文，不写日志。
- 对不存在邮箱使用一致响应和 suppressed challenge，降低账号枚举风险。
- 修改密码后撤销已有会话。

## 13. 不可破坏的项目决策

- 实验必须基于实验开始后真实执行且成功的命令，不能恢复“自动预写历史即可通过”。
- 学习进度必须同时有本地兜底和云端同步，云端失败不能静默清空本地数据。
- 删除、清空、导入或覆盖进度前必须先备份。
- Electron `appId` 不得修改，卸载和更新不得默认删除 `userData`。
- 密码、Token、数据库连接串、SMTP 授权码和验证码不得进入 Git、日志、URL 或前端存储。
- 外部 JSON 必须以 `unknown` 接收并通过类型守卫。
- 不得用 Pages 代替需要 API 的正式账号版。
- 不得绕过 `AGENTS.md` 质量门禁。
- 不得擅自覆盖用户或其他 Agent 的未提交改动。
