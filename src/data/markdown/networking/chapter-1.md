## 2.1 IP 地址基础

### IPv4 地址结构

IPv4 地址为 **32 位**，通常用点分十进制表示：\`192.168.1.1\`

每段 8 位（1 字节），取值范围 0~255。

\`\`\`
192  .  168  .  1   .  1
11000000.10101000.00000001.00000001  (二进制)
\`\`\`

### IP 地址分类（传统分类）

| 类别 | 首位 | 网络号位数 | 地址范围 | 默认子网掩码 | 用途 |
|------|------|-----------|----------|-------------|------|
| A 类 | 0 | 前 8 位 | 1.0.0.0 ~ 126.255.255.255 | 255.0.0.0 (/8) | 大型网络 |
| B 类 | 10 | 前 16 位 | 128.0.0.0 ~ 191.255.255.255 | 255.255.0.0 (/16) | 中型网络 |
| C 类 | 110 | 前 24 位 | 192.0.0.0 ~ 223.255.255.255 | 255.255.255.0 (/24) | 小型网络 |
| D 类 | 1110 | - | 224.0.0.0 ~ 239.255.255.255 | - | 多播 |
| E 类 | 11110 | - | 240.0.0.0 ~ 255.255.255.255 | - | 保留 |

> 注意：127.0.0.0/8 为回环地址（127.0.0.1），不参与网络通信。

### 私有 IP 地址（RFC 1918）

私有地址不能出现在公网，需通过 NAT 转换才能访问互联网：

| 类别 | 私有地址范围 | CIDR 表示 |
|------|-------------|-----------|
| A 类 | 10.0.0.0 ~ 10.255.255.255 | 10.0.0.0/8 |
| B 类 | 172.16.0.0 ~ 172.31.255.255 | 172.16.0.0/12 |
| C 类 | 192.168.0.0 ~ 192.168.255.255 | 192.168.0.0/16 |

## 2.2 子网掩码与 CIDR

### 子网掩码的作用

子网掩码用于区分 IP 地址中的**网络部分**和**主机部分**。

\`\`\`
IP 地址：     192.168.1.100
子网掩码：    255.255.255.0
CIDR 写法：   192.168.1.100/24

网络部分：    192.168.1
主机部分：             100
\`\`\`

### 常见子网掩码对照

| CIDR | 子网掩码 | 可用 IP 数 | 二进制网络位 |
|------|---------|-----------|-------------|
| /8 | 255.0.0.0 | 16,777,214 | 11111111.00000000.00000000.00000000 |
| /16 | 255.255.0.0 | 65,534 | 11111111.11111111.00000000.00000000 |
| /24 | 255.255.255.0 | 254 | 11111111.11111111.11111111.00000000 |
| /25 | 255.255.255.128 | 126 | ...10000000 |
| /26 | 255.255.255.192 | 62 | ...11000000 |
| /27 | 255.255.255.224 | 30 | ...11100000 |
| /28 | 255.255.255.240 | 14 | ...11110000 |

> IP 数 = 2^(32-前缀长度) - 2（减去网络地址和广播地址）

## 2.3 子网划分实战

### 问题：将 192.168.1.0/24 划分为 4 个子网

需要 4 个子网 → 需要借 2 位（2² = 4）→ 子网掩码变为 /26

\`\`\`
原始网络：192.168.1.0/24
新掩码：/26 (255.255.255.192)

子网 1: 192.168.1.0/26   范围: 192.168.1.0   - 192.168.1.63   可用: .1-.62
子网 2: 192.168.1.64/26  范围: 192.168.1.64  - 192.168.1.127  可用: .65-.126
子网 3: 192.168.1.128/26 范围: 192.168.1.128 - 192.168.1.191  可用: .129-.190
子网 4: 192.168.1.192/26 范围: 192.168.1.192 - 192.168.1.255  可用: .193-.254
\`\`\`

### 计算步骤（通用方法）

1. 确定需要多少个子网 → 计算借位数 n（2^n >= 子网数）
2. 新掩码 = 原掩码 + n
3. 块大小 = 2^(8 - 新掩码对 8 的余数)（针对 C 类便捷算法）
4. 每个子网的网络地址 = 前一个网络地址 + 块大小

### 使用 ipcalc 工具

\`\`\`bash
# 安装 ipcalc
sudo apt install ipcalc    # Debian/Ubuntu
sudo yum install ipcalc    # CentOS/RHEL

# 计算子网信息
ipcalc 192.168.1.0/24 -s 4 4 4 8

# 查看网段信息
ipcalc 192.168.1.100/26
\`\`\`

## 2.4 特殊 IP 地址

| 地址 | 用途 |
|------|------|
| 0.0.0.0 | 表示所有 IP 地址（服务器监听）或默认路由 |
| 255.255.255.255 | 受限广播（本网段广播） |
| 127.0.0.1 | 本地回环 |
| 169.254.0.0/16 | 链路本地地址（DHCP 失败时自动分配） |
| 网络号全 0 | 本网络 |
| 主机号全 0 | 网络地址 |
| 主机号全 1 | 广播地址 |

## 2.5 IPv6 简介

IPv6 地址为 **128 位**，用冒号分隔的十六进制表示：

\`\`\`
2001:0db8:0000:0000:0000:ff00:0042:8329
简写：2001:db8::ff00:42:8329
\`\`\`

### IPv6 vs IPv4

| 特性 | IPv4 | IPv6 |
|------|------|------|
| 地址长度 | 32 位 | 128 位 |
| 地址数量 | ~43 亿 | 几乎无限 |
| NAT | 普遍需要 | 不需要 |
| 配置 | DHCP/手动 | SLAAC 自动配置 |
| 安全性 | IPsec 可选 | IPsec 内置 |
| QoS | 不支持 | 支持流标签 |

### IPv6 在 Linux 中的配置

\`\`\`bash
# 查看 IPv6 地址
ip -6 addr show

# 添加 IPv6 地址
ip -6 addr add 2001:db8::1/64 dev eth0

# 测试 IPv6 连接
ping6 2001:db8::1
curl -6 https://ipv6.example.com
\`\`\`
